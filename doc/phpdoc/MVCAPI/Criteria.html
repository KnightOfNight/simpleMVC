Criteria

Description
Description | Methods (details) Constants (details)

Manage all of the criteria for a database select: which columns get selected, what sort order to use, where clause, etc.

In this class, all column names are expected to be fully qualified, e.g. <model>.<column>

    8. 

A clause takes the following form...  
  1.  array (array (<col>, <operator>, <val>))
    2. 

  2.  e.g. array (array ("model.col", Criteria::op_eq, "col val"))
    3. 

  It is an array of arrays because you can expand the clause to include multiple  subclauses.  Like so...  
  1.  array ( array (<col>, <operator>, <val>), <operator>, array (<col>, <operator>, <val>) )
    2. 

  2.  e.g. array (array ("model.col", Criteria::op_eq, "col val"), Criteria::op_or,
    3. 

  3.      array ("model.col", Criteria::op_eq, "col val 2"))
    4.
